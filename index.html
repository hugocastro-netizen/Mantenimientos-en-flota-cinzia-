Ítems

Registrar mantenimiento realizado

Actualiza directamente el km al que le toca el próximo cambio de ese ítem.

Registrar reparación / acción correctiva

Para roturas o arreglos puntuales, fuera del cronograma de mantenimiento.

Postergar mantenimiento

Suma los km que indiques al próximo cambio (deja de estar vencido), pero queda marcado como "Postergado" con el motivo, para que se sepa que no se hizo realmente.

Catálogo de mantenimientos

Acá vive la lista de ítems que existen y cada cuántos km se DEBE hacer cada uno. Cuando registrás un mantenimiento como hecho, el dashboard usa este intervalo para sugerirte el próximo km solo. Agregá uno nuevo si te falta algo de la lista.

Generar informe PDF

Genera un informe detallado por sucursal con todas las reservas del mes seleccionado.

Cargar reserva

Se puede cargar con anticipación, antes de que el viaje empiece. Cuando llegue la fecha de vuelta, va a aparecer sola en "Pendientes de cargar" para que digas si tuvo reclamo.

Cargar reporte de reserva

Una fila por reserva. Se actualiza si ya existe el mismo móvil + N° de reserva.

Conectar tus datos reales

Hay dos partes: lectura (traer tus datos de Sheets) y escritura (guardar lo que registrás desde los botones).

1) Lectura — tus hojas de Sheets

  1. Compartí cada hoja (inspecciones, mantenimiento) como "Cualquiera con el enlace — Lector".
  2. Copiá el ID de cada una desde la URL, entre /d/ y /edit.
  3. Pegalos en el bloque CONFIG.SHEETS al inicio del <script>, junto con el nombre de cada pestaña.
  4. Revisá que CONFIG.COLUMNS coincida con tus headers reales.
  5. Cambiá USE_MOCK_DATA a false.

2) Escritura — guardar mantenimientos, reparaciones y neumáticos

Esto necesita un pequeño backend gratuito (Google Apps Script). Te lo dejo en el archivo registro-acciones-api.gs: lo pegás en Apps Script, lo "implementás" como aplicación web, y la URL que te da va en CONFIG.API_URL.

const CONFIG = {
  USE_MOCK_DATA: false,
  API_URL: "https://script.google.com/macros/s/AKfycb.../exec",
  SHEETS: { inspecciones: {...}, mantenimiento: {...} },
  ...
};

Si dejás API_URL vacío, los botones siguen funcionando pero solo localmente (no se guarda para el resto del equipo).
